# Reproducibility guide

## Требования

- Python 3.10+
- Linux / macOS / Windows
- Желательно GPU для ускорения текстовых частей пайплайна, но базово код может работать и на CPU

## Данные

В папке `data/` должны лежать файлы:

- `users.csv`
- `editions.csv`
- `authors.csv`
- `genres.csv`
- `book_genres.csv`
- `interactions.csv`
- `targets.csv`
- `candidates.csv`

Если данные лежат в другом месте, можно поправить константу `DATA_DIR = 'data/'` в скриптах.

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск

### Вариант 1: полный пайплайн

```bash
bash run.sh
```

### Вариант 2: по шагам

```bash
python src/v10.py
python src/v11.py
python src/blend.py
```

## Ожидаемые артефакты

После выполнения появятся:

- `submission888.csv`
- `submission (4) (1).csv`
- `blend_2_anchor_top7.csv`

Финальным сабмитом является `blend_2_anchor_top7.csv`.

## Фиксация случайности

В коде используются фиксированные seed:

- `np.random.seed(42)`
- `random_state=42` в ряде моделей/алгоритмов

Это повышает воспроизводимость результатов при одинаковых данных и совместимых версиях библиотек.

## Практические замечания

- Некоторые шаги могут занимать заметное время.
- Для конкурсного воспроизведения лучше использовать окружение, близкое к тому, в котором решение отлаживалось.
- Если полное численное совпадение не достигается, структура решения и качество должны оставаться близкими при тех же данных и зависимостях.
