## О проекте

Это репозиторий моего конкурсного решения для хакатона Академии ИИ **«По страницам»**.
Хакатон был посвящён задаче рекомендательных систем: нужно было ранжировать кандидатов и находить баланс между релевантностью, разнообразием и устойчивостью решения. По официальной странице соревнование проходило с **3 февраля по 18 февраля 2026 года** и было ориентировано на школьников и студентов СПО. 

Моё решение заняло **2 место в категории «Школьники»**.

## Что внутри

Решение построено как **ансамбль нескольких рекомендательных представлений** с последующим **learning-to-rank** и финальным **blending / reranking**:

- матрица пользователь–книга с весами по типам взаимодействий;
- несколько вариантов **SVD**-эмбеддингов;
- две конфигурации **ALS**;
- **BPR** для pairwise preference learning;
- **EASE** для item-item сигналов;
- **item2vec / PPMI + SVD** для ко-оккурентных представлений;
- набор признаков по авторам, жанрам, языкам, годам и текстам;
- **CatBoostRanker** как основная ранжирующая модель;
- **diversity reranking** для улучшения разнообразия рекомендаций;
- финальный **blend** нескольких сабмитов через RRF-подобное объединение.

Идея проекта — не полагаться на одну модель, а собрать сильный и устойчивый ансамбль из разных сигналов и затем аккуратно пересобрать итоговый ранжированный список.

## Как запустить

### 1. Подготовить окружение

Рекомендуемая версия Python: **3.10+**

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Положить данные

Помести в папку `data/` исходные файлы соревнования:

- `users.csv`
- `editions.csv`
- `authors.csv`
- `genres.csv`
- `book_genres.csv`
- `interactions.csv`
- `targets.csv`
- `candidates.csv`

### 3. Запустить пайплайн

```bash
bash run.sh
```

или по шагам:

```bash
python src/v10.py
python src/v11.py
python src/blend.py
```

## Что получится на выходе

После выполнения пайплайна в корне появятся:

- `submission888.csv` — результат базового ансамбля;
- `submission (4) (1).csv` — основной результат после reranking;
- `blend_2_anchor_top7.csv` — финальный blended submission.

## Коротко про модели

### `src/v10.py`
Базовый ансамбль представлений и признаков. Формирует один из сильных опорных сабмитов.

### `src/v11.py`
Основной пайплайн. Добавляет более сильную сборку признаков и **diversity reranking**, который помогает улучшать качество итогового списка не только по релевантности, но и по разнообразию.

### `src/blend.py`
Финальный merge двух сабмитов. Используется RRF-подобная схема с якорением верхней части ранжирования из лучшего решения.

## Почему решение получилось сильным

1. **Разные сигналы дополняют друг друга.**
   Коллаборативные, item-item и семантические признаки покрывают разные сценарии поведения пользователя.

2. **Ranking поверх candidate set.**
   Вместо одной эвристики используется полноценная LTR-модель на богатом наборе признаков.

3. **Отдельная работа с diversity.**
   Для книжных рекомендаций важно не только «попасть в жанр», но и не повторять слишком однообразный список.

4. **Финальный blend.**
   Даже хорошие сабмиты часто дают лучший результат после аккуратного объединения.
